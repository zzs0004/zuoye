# 系统设计文档

> 高校实验室设备借用管理系统  
> 版本：v2.0 | 日期：2026-06-19（修订版，与代码一致）

## 1. 架构设计

### 1.1 整体架构

```
┌──────────────────────────────────────┐
│              浏览器（前端）            │
│        HTML + CSS + JavaScript       │
│           单页应用 (SPA)              │
└──────────────┬───────────────────────┘
               │  HTTP/REST API (Bearer Token 认证)
┌──────────────▼───────────────────────┐
│           Flask Web 服务器            │
│        Python 3.x + Flask 3.x        │
│    路由层 → 业务逻辑层 → 数据访问层    │
└──────────────┬───────────────────────┘
               │  sqlite3
┌──────────────▼───────────────────────┐
│              SQLite 数据库            │
│              lab.db                  │
└──────────────────────────────────────┘
```

### 1.2 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | 原生 HTML/CSS/JS | 轻量，无需框架，单文件部署 |
| 后端 | Flask 3.x | 轻量级 Web 框架，适合小型项目 |
| 数据库 | SQLite | 零配置，文件型数据库，适合单机部署 |
| 认证 | Bearer Token（用户ID） | 简洁的无状态认证方案 |
| API 风格 | RESTful | 前后端分离，标准化接口 |
| 密码加密 | Werkzeug scrypt | Flask 内置安全哈希 |

### 2.2 表结构

**users（用户表）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 用户ID |
| username | TEXT | UNIQUE, NOT NULL | 用户名 |
| password | TEXT | NOT NULL | 密码（scrypt哈希） |
| role | TEXT | DEFAULT 'student', CHECK IN ('student','admin') | 角色 |
| phone | TEXT | | 手机号 |
| created_at | TEXT | DEFAULT datetime('now','localtime') | 注册时间 |

**equipment（设备表）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 设备ID |
| name | TEXT | NOT NULL | 设备名称 |
| model | TEXT | | 型号规格 |
| total_qty | INTEGER | NOT NULL DEFAULT 1 | 总数量 |
| available_qty | INTEGER | NOT NULL DEFAULT 1 | 可借数量 |
| location | TEXT | | 存放位置 |
| status | TEXT | DEFAULT 'available', CHECK IN ('available','unavailable') | 状态 |
| admin_name | TEXT | | 管理员 |
| created_at | TEXT | DEFAULT datetime('now','localtime') | 录入时间 |

**borrow_records（借用记录表）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 记录ID |
| user_id | INTEGER | FK → users.id | 借用人 |
| equipment_id | INTEGER | FK → equipment.id | 借用设备 |
| equipment_name | TEXT | NOT NULL | 设备名称（冗余） |
| borrow_time | TEXT | DEFAULT datetime('now','localtime') | 借用时间 |
| expected_return | TEXT | NOT NULL | 预计归还时间 |
| actual_return | TEXT | | 实际归还时间 |
| status | TEXT | DEFAULT 'borrowed', CHECK IN ('borrowed','returned','overdue') | 状态 |

> 借用流程无需审批，学生提交后即时生效。系统自动校验逾期记录和设备库存。

**notifications（通知表）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 通知ID |
| user_id | INTEGER | FK → users.id | 接收用户 |
| borrow_id | INTEGER | FK → borrow_records.id（可空） | 关联借用记录 |
| content | TEXT | NOT NULL | 提醒内容 |
| is_read | INTEGER | DEFAULT 0 | 0=未读, 1=已读 |
| created_at | TEXT | DEFAULT datetime('now','localtime') | 创建时间 |

## 3. API 接口设计

### 3.1 认证相关（无需登录）

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/register` | 无 | 用户注册（默认角色student） |
| POST | `/api/login` | 无 | 用户登录，返回Bearer Token |

### 3.2 用户管理（需登录）

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| GET | `/api/user/info` | 全部 | 获取当前用户信息 |
| PUT | `/api/user/password` | 全部 | 修改密码（需旧密码） |

### 3.3 设备管理

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| GET | `/api/equipment` | 全部 | 设备列表（支持 q/status 筛选） |
| GET | `/api/equipment/<id>` | 全部 | 单个设备详情 |
| POST | `/api/equipment` | 管理员 | 添加设备 |
| PUT | `/api/equipment/<id>` | 管理员 | 编辑设备（自动计算可用量） |
| DELETE | `/api/equipment/<id>` | 管理员 | 删除设备（需无未归还记录） |

### 3.4 借用管理

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| POST | `/api/borrow` | 学生 | 借用设备（自动校验逾期+库存） |
| GET | `/api/borrow/records` | 全部 | 查询借用记录（学生看自己，管理员看全部） |
| POST | `/api/borrow/<id>/return` | 全部 | 归还设备（自动判定逾期） |

### 3.5 通知提醒

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| GET | `/api/notifications` | 全部 | 当前用户通知列表 |
| GET | `/api/notifications/unread-count` | 全部 | 未读通知数 |
| POST | `/api/notifications/<id>/read` | 全部 | 标记单条已读 |
| POST | `/api/notifications/send` | 管理员 | 手动发送提醒（支持指定用户/记录/批量） |

### 3.6 数据统计

| 方法 | 路径 | 角色 | 说明 |
|------|------|------|------|
| GET | `/api/statistics/equipment` | 管理员 | 设备借用频次统计 |
| GET | `/api/statistics/users` | 管理员 | 用户借用行为统计 |
| GET | `/api/statistics/export` | 管理员 | CSV导出设备统计 |


## 5. 关键业务逻辑

### 5.1 借用申请流程

```
学生 POST /api/borrow {equipment_id, expected_return}
  → check_overdue() 更新逾期状态 + 自动生成通知
  → 校验：用户是否有逾期记录 → 有则驳回
  → 校验：设备 available_qty > 0 → 否则驳回
  → 事务：INSERT borrow_records (status='borrowed')
         + UPDATE equipment SET available_qty-=1
         + 若 available_qty=0 → status='unavailable'
  → 返回："借用成功"
```

### 5.2 逾期检测与通知生成

```
check_overdue()
  → 查询所有 status='borrowed' 且 expected_return < now 的记录
  → UPDATE 这些记录 status='overdue'
  → 对每条新逾期记录：检查是否已存在逾期通知
  → 不存在则 INSERT INTO notifications
```

### 5.3 归还流程

```
POST /api/borrow/<id>/return
  → 权限检查：学生只能还自己的，管理员可还任何人的
  → 状态检查：已归还不能重复归还
  → 判定逾期：actual_return > expected_return → status='overdue'
                                         否则 → status='returned'
  → UPDATE borrow_records + UPDATE equipment (available_qty+1)
  → 若 available_qty>0 → status='available'
```
