## 1. 系统简介

高校实验室设备借用管理系统是一个基于 Web 的轻量级管理工具，支持设备信息维护、在线借用申请、审批流程管理等功能。

## 2. 安装部署

### 2.1 环境要求

- Python 3.8 或以上版本
- pip 包管理器
- 现代浏览器（Chrome / Firefox / Edge）

### 2.2 安装步骤

```bash
# 1. 进入项目目录
cd C:\Users\14396\lab-equipment-system

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动服务
python app.py
```

### 2.3 访问系统

浏览器打开：**http://localhost:5000**

### 2.4 默认账号

| 角色 | 用户名 | 密码 |
|------|--------|------|
| 管理员 | admin | admin123 |

> ⚠️ 首次使用请立即修改管理员密码

## 3. 操作指南

### 3.1 登录系统

1. 打开浏览器访问 http://localhost:5000
2. 输入用户名和密码
3. 点击「登录」

### 3.2 设备管理（管理员）

**添加设备：**
1. 点击「设备管理」菜单
2. 点击「添加设备」按钮
3. 填写设备名称、编号、型号
4. 点击「保存」

**编辑设备：**
1. 在设备列表中找到目标设备
2. 点击「编辑」按钮
3. 修改信息后点击「保存」

**删除设备：**
1. 在设备列表中找到目标设备
2. 点击「删除」按钮
3. 确认删除

### 3.3 借用设备（用户）

1. 在设备列表中浏览可用设备
2. 点击目标设备的「借用」按钮
3. 确认借用信息
4. 等待管理员审批

### 3.4 审批管理（管理员）

1. 点击「借用管理」菜单
2. 查看待审批列表
3. 点击「通过」或「拒绝」
4. 可选：填写审批意见

### 3.5 归还设备

1. 在借用记录中找到待归还记录
2. 点击「归还」按钮
3. 确认归还

## 4. 设备状态说明

| 状态 | 含义 | 可执行操作 |
|------|------|------------|
| 可用 (available) | 设备空闲可借用 | 借用 |
| 已借出 (borrowed) | 设备已被借出 | 归还 |
| 维修中 (maintenance) | 设备维护中 | 不可借用 |

## 5. 借用状态说明

| 状态 | 含义 |
|------|------|
| 待审批 (pending) | 申请已提交，等待管理员处理 |
| 已通过 (approved) | 管理员已批准，设备已借出 |
| 已拒绝 (rejected) | 管理员已拒绝申请 |
| 已归还 (returned) | 设备已归还 |

## 6. 常见问题

### Q: 忘记密码怎么办？
联系管理员重置密码。

### Q: 启动报错 "Address already in use"？
5000 端口被占用，关闭其他程序或修改 app.py 中的端口号。

### Q: 数据会丢失吗？
所有数据存储在 `lab.db` 文件中，备份此文件即可备份全部数据。
